# 详情页返回后保留“最新”分类修复方案

> 目标：用户在首页选择顶部“最新”分类，进入任一视频详情页后点击左上角返回，必须回到进入前的“最新”分类、同一页数据和原滚动位置；不能回落到“热门”。
>
> 本文供 DeepSeek 实施。当前工作区已有未提交修改，必须增量合并，禁止 `git reset --hard`、`git checkout --` 或整文件覆盖。

## 0. 二次排查后的紧急更正（2026-07-13）

分类接口已确认：

```json
[
  { "type_id": 1, "type_name": "热门" },
  { "type_id": 0, "type_name": "最新" }
]
```

因此“最新”的合法分类 ID 就是数字 `0`。分类 ID 的校验规则必须是 `>= 0`，不能是 `> 0`，也不能用 `if (!typeId)` 判断是否缺失。只有页码 `page` 必须 `> 0`。

当前源码中还有三个必现问题，必须一次全部修改：

1. `src/views/VideoDetailView.vue` 使用 `parsePositiveInt()` 解析 `returnTypeId`，并且还有 `if (!typeId ...)`，所以 `returnTypeId=0` 一定被丢弃。
2. `src/utils/home-detail-return.ts` 要求 `value.typeId > 0`，即使详情页保存了 `0`，首页消费快照时仍会判定无效。
3. `src/views/HomeView.vue` 使用 `sessionData.activeTypeId || 1`，会把合法的 `0` 再次替换为热门 `1`。

这三处任何一处没改，“最新 → 详情 → 返回”都不可能正确。

## 1. 根因

当前“列表 → 详情 → 返回”已经部分建立了来源分类快照，但对分类 ID `0` 的语义处理错误，导致快照在详情解析、快照消费和首页 session 恢复三个阶段被丢弃或改写。

### 1.1 当前链路

1. `src/components/VideoList.vue` 已经把当前 `typeId`、页码和滚动位置写入详情 query。从“最新”进入时，正确 URL 应包含 `returnTypeId=0`。
2. `src/views/VideoDetailView.vue` 错误地使用只允许 `> 0` 的 `parsePositiveInt()` 解析 `returnTypeId`，随后又使用 `!typeId` 判空，因此不会保存最新返回快照。
3. `src/utils/home-detail-return.ts` 再次要求 `typeId > 0`，会把所有 `typeId=0` 快照当作无效数据删除。
4. `src/views/HomeView.vue` 的 `restoreSessionData()` 使用 `activeTypeId || 1`，会把 session 中的最新 `0` 改成热门 `1`。

这是同一个“零值被当成空值”错误在三个阶段的重复出现，不是分类接口顺序、`KeepAlive` 或 `router.back()` 本身错误。

## 2. 目标行为

| 场景 | 期望结果 |
| --- | --- |
| 热门列表进入详情并返回 | 回热门、原页码、原滚动位置。 |
| **最新列表进入详情并返回** | **回最新、原页码、原滚动位置，不显示热门。** |
| 一级分类含二级分类时，从二级分类进入详情并返回 | 恢复对应一级展开态、二级选中态和列表。 |
| 列表缓存仍有效 | 直接恢复缓存，不重复请求，也不闪现热门列表。 |
| 列表缓存已失效/内存被清理 | 用进入详情前的分类 ID 与页码重新拉取正确列表，不回退到热门。 |
| 从外部链接直接进入 `/video/:id` | 维持现有浏览器返回逻辑；没有来源上下文时不要强行跳首页。 |
| 用户用浏览器后退而非详情页按钮 | 不报错；页面至少保留现有缓存恢复逻辑。详情页左上角返回必须保证本方案的精确恢复。 |

## 3. 改造原则

1. **分类状态是页面级状态，不是 URL 字符串 `from=/` 能表达的。** 需要显式传递结构化快照。
2. 详情页点击返回前，把来源快照写入仅当前标签页有效的 `sessionStorage`；然后继续使用 `router.back()`，保留自然历史栈，不额外 `push` 一个首页记录。
3. 首页激活时，**先**恢复已有缓存，**再**消费本次详情返回快照并覆盖缓存恢复结果；返回快照优先级最高。
4. 首页根据快照确定分类后，再决定“恢复该分类缓存”或“按该分类重新请求”。不能先按热门请求、再切最新。
5. 只传递类别 ID、页码、滚动位置和时间戳；不把整份视频列表塞进 URL 或 sessionStorage 新键。
6. 所有 sessionStorage 数据必须验证类型、类别是否存在、时效是否有效；无效数据直接丢弃并走原有逻辑。

| 文件 | 改动 |

## 5. 统一的详情返回快照工具

继续使用现有 `src/utils/home-detail-return.ts`，不要再创建另一个快照工具，也不要让 `HomeView`、`VideoList`、`VideoDetailView` 各自硬编码 sessionStorage key 和字段名。

### 5.1 数据类型

```ts
export const HOME_DETAIL_RETURN_STATE_KEY = 'homeDetailReturnState'
export const HOME_DETAIL_RETURN_TTL = 10 * 60 * 1000

export interface HomeDetailReturnState {
  source: 'home'
  typeId: number
  subTypeId: number | null
  expandedTypeId: number | null
  page: number
  scrollPosition: number
  createdAt: number
}

export interface HomeDetailReturnContext {
  typeId: number
  subTypeId: number | null
  expandedTypeId: number | null
  page: number
}
```

说明：

- `typeId` 是当前真正用于列表请求/选中的 ID；从“最新”进入时必须是“最新”对应的 ID，不能写死 `1` 或热门 ID。
- `subTypeId` 和 `expandedTypeId` 用于二级分类场景；没有时固定为 `null`，不要使用 `undefined`、`0` 或字符串 `'null'`。
- `page` 记录列表当前页，最小为 1。
- `scrollPosition` 由点击时的 `window.scrollY || document.documentElement.scrollTop` 获取。
- `createdAt` 用于防止刷新、长时间停留详情后消费过期状态。

### 5.2 工具方法

至少实现：

```ts
export const saveHomeDetailReturnState = (state: HomeDetailReturnState): void => {
  sessionStorage.setItem(HOME_DETAIL_RETURN_STATE_KEY, JSON.stringify(state))
}

export const consumeHomeDetailReturnState = (): HomeDetailReturnState | null => {
  const raw = sessionStorage.getItem(HOME_DETAIL_RETURN_STATE_KEY)
  sessionStorage.removeItem(HOME_DETAIL_RETURN_STATE_KEY)

  if (!raw) return null

  try {
    const value = JSON.parse(raw) as Partial<HomeDetailReturnState>
    const isValid =
      value.source === 'home' &&
      Number.isInteger(value.typeId) && (value.typeId as number) >= 0 &&
      Number.isInteger(value.page) && (value.page as number) > 0 &&
      Number.isFinite(value.scrollPosition) &&
      Number.isFinite(value.createdAt) &&
      Date.now() - (value.createdAt as number) <= HOME_DETAIL_RETURN_TTL

    if (!isValid) return null

    return {
      source: 'home',
      typeId: value.typeId as number,
      subTypeId: Number.isInteger(value.subTypeId) ? (value.subTypeId as number) : null,
      expandedTypeId: Number.isInteger(value.expandedTypeId)
        ? (value.expandedTypeId as number)
        : null,
      page: value.page as number,
      scrollPosition: value.scrollPosition as number,
      createdAt: value.createdAt as number,
    }
  } catch {
    return null
  }
}
```

要求：

- `consume` 必须“读后即删”，确保一次返回状态不会影响下一次普通进入首页。
- `sessionStorage` 不可用或 JSON 解析失败时不得阻断正常返回；捕获错误后返回 `null`。
- 不使用 `localStorage`：这是一次详情导航的临时状态，不应跨浏览器标签页或长期保留。

当前组件使用运行时 `defineProps`，新增可选 prop 时应使用 `PropType`：

```ts
import type { PropType } from 'vue'

homeReturnContext: {
  type: Object as PropType<HomeDetailReturnContext>,
  default: undefined,
},
```

在 `handleVideoClick` 中，以对象形式跳转并仅在上下文存在时添加下列 query：

```ts
const handleVideoClick = (videoId: number) => {
  const context = props.homeReturnContext
  const scrollPosition = Math.max(
    0,
    Math.round(window.scrollY || document.documentElement.scrollTop),
  )

  router.push({
    name: 'videoDetail',
    params: { id: videoId },
    query: context
      ? {
          from: 'home',
          returnTypeId: String(context.typeId),
          returnSubTypeId: context.subTypeId === null ? '' : String(context.subTypeId),
          returnExpandedTypeId:
            context.expandedTypeId === null ? '' : String(context.expandedTypeId),
          returnPage: String(Math.max(1, context.page)),
          returnScroll: String(scrollPosition),
        }
      : { from: props.returnPath },
  })
}
```

`returnScroll` 只包含点击瞬间的非负整数坐标，用于这一次详情返回；它不是敏感数据，也不包含完整列表。不要把完整视频对象、token、用户数据或任意 JSON 放进 URL。

## 7. 详情页返回：保存 pending 状态后再回退

### 7.1 解析 query

在 `src/views/VideoDetailView.vue` 中分开解析“分类 ID”和“页码”。只在 `route.query.from === 'home'` 时返回有效上下文；`returnTypeId` 必须是非负整数（允许 `0`），`returnPage` 必须是正整数。

```ts
const parseCategoryId = (value: unknown): number | null => {
  const source = Array.isArray(value) ? value[0] : value
  if (typeof source !== 'string' || !/^\d+$/.test(source)) return null
  const parsed = Number(source)
  return Number.isSafeInteger(parsed) && parsed >= 0 ? parsed : null
}

const parsePositiveInt = (value: unknown): number | null => {
  const source = Array.isArray(value) ? value[0] : value
  if (typeof source !== 'string' || !/^\d+$/.test(source)) return null
  const parsed = Number(source)
  return Number.isSafeInteger(parsed) && parsed > 0 ? parsed : null
}

const parseNullableCategoryId = (value: unknown): number | null => {
  const source = Array.isArray(value) ? value[0] : value
  if (source === '' || source === undefined) return null
  return parseCategoryId(source)
}

const getHomeReturnContext = (): (HomeDetailReturnContext & { scrollPosition: number }) | null => {
  if (route.query.from !== 'home') return null

  const typeId = parseCategoryId(route.query.returnTypeId)
  const page = parsePositiveInt(route.query.returnPage)
  const scrollPosition = parseNonNegativeInt(route.query.returnScroll)
  if (typeId === null || page === null || scrollPosition === null) return null

  return {
    typeId,
    page,
    subTypeId: parseNullableCategoryId(route.query.returnSubTypeId),
    expandedTypeId: parseNullableCategoryId(route.query.returnExpandedTypeId),
    scrollPosition,
  }
}
```

并补充：

```ts
const parseNonNegativeInt = (value: unknown): number | null => {
  const source = Array.isArray(value) ? value[0] : value
  if (typeof source !== 'string' || !/^\d+$/.test(source)) return null
  const parsed = Number(source)
  return Number.isSafeInteger(parsed) && parsed >= 0 ? parsed : null
}
```

### 7.2 改造 `goBack`

把现有只调用 `router.back()` 的 `goBack` 修改为：

```ts
const goBack = () => {
  if (isPlaying.value) stopPlayback()

  const context = getHomeReturnContext()
  if (context) {
    saveHomeDetailReturnState({
      source: 'home',
      ...context,
      createdAt: Date.now(),
    })
  }

  router.back()
}
```

详情页还需严格解析非负整数 `returnScroll`，并把它写入 `HomeDetailReturnState`；不要让 `VideoDetailView` 读取全局 `homeScrollPosition`。这样每个详情路由都携带完整、可审计的来源快照，不会被其他列表导航覆盖。

`router.back()` 保持不变：历史记录仍为“首页 → 详情 → 首页（回退）”，不会出现“按一次返回又回到详情页”的额外历史记录。

## 8. 首页激活：先消费返回快照，再恢复正确分类

### 8.0 先修复通用 session 恢复的零值问题

`restoreSessionData()` 中不能用 `||` 为数值分类提供默认值，必须使用空值合并 `??`：

```ts
activeTypeId.value = sessionData.activeTypeId ?? 1
activeSubTypeId.value = sessionData.activeSubTypeId ?? null
expandedTypeId.value = sessionData.expandedTypeId ?? null
```

`||` 会把 `0`、`''`、`false` 等 falsy 值都当成“没有值”；`??` 只在 `null` 或 `undefined` 时使用默认值，因此可以保留合法的“最新” ID `0`。

### 8.1 应用快照函数

在 `src/views/HomeView.vue` 新增 `applyHomeDetailReturnState`，只负责验证快照与当前分类树匹配，并设置状态：

```ts
const isKnownTypeId = (typeId: number) =>
  typesList.value.some(
    (type) => type.type_id === typeId || type.child?.some((child) => child.type_id === typeId),
  )

const applyHomeDetailReturnState = (state: HomeDetailReturnState): boolean => {
  if (!isKnownTypeId(state.typeId)) return false
  if (state.subTypeId !== null && !isKnownTypeId(state.subTypeId)) return false
  if (state.expandedTypeId !== null && !typesList.value.some((type) => type.type_id === state.expandedTypeId)) {
    return false
  }

  activeTypeId.value = state.typeId
  activeSubTypeId.value = state.subTypeId
  expandedTypeId.value = state.expandedTypeId
  currentPage.value = state.page
  return true
}
```

验证更严格的版本应确认：`subTypeId` 确实属于 `expandedTypeId` 的 child，或它就是当前 `typeId`。不能让 query 中任意合法的两个分类 ID 拼成错误 UI 状态。

### 8.2 调整 `onActivated` 顺序

在现有 `HomeView.onActivated` 的非首次激活分支中，执行顺序必须改为：

```text
1. ensureTouristSessionOnActivated()
2. restoreSessionData()                     // 恢复一般缓存
3. consumeHomeDetailReturnState()           // 读后即删
4. applyHomeDetailReturnState(pending)      // 若有效，覆盖一般缓存的热门状态
5. 根据最终 activeTypeId 恢复对应 tab cache；缓存无效则按 state.page 请求
6. nextTick/requestAnimationFrame 后恢复 state.scrollPosition
7. saveSessionData()                        // 把最终、正确的最新状态写回缓存
```

关键点：**绝对不能在第 5 步先按 `restoreSessionData()` 恢复出的热门状态渲染/请求，再在异步回调里切成最新。** 否则会产生热门闪现、旧请求竞态和“返回后仍是热门”的问题。

建议将现有“恢复缓存或请求列表”的逻辑抽成一个函数：

```ts
const restoreOrFetchActiveTab = async (preferredPage = 1) => {
  const tabState = tabStates.value[activeTypeId.value]
  if (shouldUseCache(tabState)) {
    restoreTabFromCache(activeTypeId.value, tabState)
    return
  }

  const firstTypeId = getFirstTypeId()
  const tid = activeTypeId.value === firstTypeId ? VIDEO_CATEGORIES.ALL : activeTypeId.value
  await fetchRecommendVideosData(preferredPage, tid)
}
```

详情返回时调用 `restoreOrFetchActiveTab(returnState.page)`；普通激活时保留当前默认参数。发请求前继续使用现有 `videoListRequestSeq` 竞态保护。

只有在快照被成功应用后，才能使用快照中的页码和滚动位置：

```ts
const returnState = consumeHomeDetailReturnState()
const appliedReturnState =
  returnState && applyHomeDetailReturnState(returnState) ? returnState : null

const returnPage = appliedReturnState?.page ?? 1
await restoreOrFetchActiveTab(returnPage)

if (appliedReturnState) {
  await nextTick()
  requestAnimationFrame(() =>
    window.scrollTo({ top: appliedReturnState.scrollPosition, behavior: 'auto' }),
  )
}
```

不能在快照校验失败后仍使用其 `page`，否则会用首页当前分类加上无效快照页码发起错误请求。

滚动恢复应只使用本次 `returnState.scrollPosition`，并在列表恢复/请求完成、DOM 更新后执行：

```ts
await nextTick()
requestAnimationFrame(() => window.scrollTo({ top: returnState.scrollPosition, behavior: 'auto' }))
```

删除 `HomeView` 中现有的两段 `homeScrollPosition` 读取/`setTimeout` 恢复逻辑，并统一改由 `HomeDetailReturnState.scrollPosition` 恢复。不得让旧键定时器与新快照定时器同时执行。

## 9. 兼容与边界

- 从搜索、收藏、观看记录等非首页入口进入详情时，没有 `from=home`/返回快照，详情页维持普通 `router.back()`；不应错误应用首页“最新”状态。
- 用户直接打开或刷新详情 URL 时，query 可能不完整，解析失败即不写 pending 状态。
- 返回快照中的分类已被服务端删除、ID 不在当前 `typesList` 中、超过 10 分钟或 JSON 损坏时，丢弃快照并走现有首页恢复，不能抛错或卡在 loading。
- 连续路径“最新 A → 详情 A → 推荐 B → 返回”必须定义清楚：推荐 B 没有首页上下文时只回详情 A；从详情 A 再返回才回最新。不能把详情推荐误当作首页来源。
- 仅为本次详情返回设置 `sessionStorage`，不要更改游客登录、广告缓存、封面缓存、`lastActiveTabId` 的含义。

## 10. 验收测试

### 手工回归

1. 清空首页会话缓存，进入首页，点击“最新”，确认 `activeTypeId === 0`。
2. 选择最新列表第一、第二页的任意视频进入详情，点击左上角返回；必须回到“最新”、原页码和原滚动位置。
3. 进入详情后检查 URL，必须包含 `from=home&returnTypeId=0`；如果是 `returnTypeId=1` 或没有该参数，说明 `VideoList` 透传仍未正确。
4. 在返回过程中观察：不得先出现热门列表后再切最新，也不得额外发出热门请求。
5. 从热门进入详情返回，仍回热门。
