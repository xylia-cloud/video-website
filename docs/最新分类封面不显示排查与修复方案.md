# 首页“最新”分类封面不显示：排查结论与修复方案

> 目标：修复首页顶部“最新”分类的视频列表有标题、有卡片但封面不显示的问题，并避免同类旧图片域名再次造成空白封面。
>
> 本文根据用户提供的 `list[1]` 数据、当前前端代码和 2026-07-13 的实际图片地址响应编写，供 DeepSeek 实施。当前工作区已存在 `src/api/core/image.ts` 与 `src/views/HomeView.vue` 的未提交修改；实施时必须在其基础上增量合并，**不得使用 reset、checkout 或覆盖文件的方式还原他人改动**。

## 1. 已确认的根因

这不是“最新”列表没有返回封面字段，也不是 Vant 懒加载本身没有触发。

用户提供的记录中：

```json
{
  "vod_id": 3139277,
  "vod_pic": "https://test.jiji8.cc/upload/vod/20260712-1/14bda738f4d86aeb7e88004fd5d7f3f9.webp",
  "vod_pic_thumb": "https://test.jiji8.cc/"
}
```

现有首页的处理链路为：

```text
HomeView.processVideoData
  └─ item.vod_pic || item.vod_pic_thumb || item.coverUrl
       └─ getFullImageUrl(...)
            └─ 绝对 http(s) 地址直接原样返回
                 └─ VideoList 的 <img v-lazy="item.coverUrl"> 请求该地址
```

因为 `vod_pic` 非空，代码正确地优先使用了它，并不会使用 `vod_pic_thumb`。但该文件地址实际响应如下：

| 验证地址 | 实际结果 | 结论 |
| --- | --- | --- |
| `https://test.jiji8.cc/upload/vod/20260712-1/14bda738f4d86aeb7e88004fd5d7f3f9.webp` | `404 Not Found`，`Content-Type: text/html` | 接口下发的测试域名无法取得图片 |
| `https://jiji1.tv/upload/vod/20260712-1/14bda738f4d86aeb7e88004fd5d7f3f9.webp` | `200 OK`，`Content-Type: image/webp`，`Content-Length: 35528` | 同一路径在正式图片域名存在且可用 |
| `https://test.jiji8.cc/`（`vod_pic_thumb` 的值） | `200 OK`，但 `Content-Type: text/html` | 这是站点首页，不是图片，不能作为缩略图回退地址 |

因此根因是：**“最新”数据中的 `vod_pic` 使用了失效/错误的 `test.jiji8.cc` 主机名；真实图片位于 `jiji1.tv` 的相同路径。**

其他分类能显示，说明它们返回的封面地址大概率已经是可访问域名或相对路径。不要据此假设“最新”要使用另一套渲染组件；首页所有分类都复用 `HomeView.processVideoData` 和 `VideoList.vue`。

## 2. 当前代码为何没有自行恢复

### 2.1 URL 工具只拼接相对路径，不规范化绝对地址

`src/api/core/image.ts` 的 `getFullImageUrl` 对 `http` 开头的地址直接返回：

```ts
if (path.startsWith('http')) {
  return path
}
```

所以 `https://test.jiji8.cc/...` 会原样进入浏览器请求，前端不会替换成可访问的 `https://jiji1.tv/...`。

### 2.2 `vod_pic_thumb` 不是有效备用封面

虽然映射逻辑有 `vod_pic || vod_pic_thumb`，但 `||` 只能判断字符串是否为空，不能判断内容是否为图片。样例中的 `vod_pic_thumb` 为站点根地址，即使拿它回退，也会返回 HTML 而非图片。

### 2.3 图片错误事件只显示深色背景

`src/components/VideoList.vue` 的 `handleImageError` 目前只设置图片元素背景色和 `alt` 文本。封面容器原本也是深色背景，因此用户看到的是“没有封面”的空卡片，没有“封面暂不可用”的明确兜底提示，也无法从页面上判断是后端 URL 错误。

## 3. 修复目标与优先级

| 优先级 | 改动 | 目的 |
| --- | --- | --- |
| P0 | 后端/CMS 输出正确的 `jiji1.tv` 图片主机名 | 从源头消除错误数据，所有客户端均受益 |
| P0 | 前端在统一图片解析处白名单规范化旧域名 | 已发布的历史数据无需等待数据库完全回填，也不会再显示空白 |
| P1 | 过滤“域名根路径”这类伪缩略图 | 避免 `vod_pic_thumb: https://test.jiji8.cc/` 被当成封面 |
| P1 | 图片加载失败时展示可辨识占位状态 | 单条源图真的不存在时，页面不出现无意义黑块 |
| P2 | 单元/组件测试与线上监控 | 防止域名错误再次上线且难以发现 |

## 4. 后端/数据源修复（必须执行）

前端兼容只能修复已知旧域名，正确做法仍是让视频接口直接返回正式可访问的图片 URL。

### 4.1 接口输出规则

视频接口返回字段应满足：

```json
{
  "vod_pic": "https://jiji1.tv/upload/vod/20260712-1/14bda738f4d86aeb7e88004fd5d7f3f9.webp",
  "vod_pic_thumb": ""
}
```

要求：

- `vod_pic` 使用公开可访问的正式域名 `https://jiji1.tv`，或只返回 `/upload/...` 相对路径；不要返回 `test.jiji8.cc`。
- 只有确实存在缩略图文件时才输出 `vod_pic_thumb`；没有缩略图必须返回空字符串/`null`，不能把站点根地址当作缩略图。
- 保留 `/upload/vod/...` 文件路径、查询参数和 hash，不要在迁移时改变文件名。
- API 变更后抽查“最新”和其他分类至少各 20 条：封面 URL 应返回 2xx 且 `Content-Type` 是 `image/*`。

### 4.2 历史数据回填原则

如果 CMS/数据库中存储的是完整 `https://test.jiji8.cc/upload/...` URL，需要由拥有数据库权限的后端人员执行有备份、可回滚的迁移：

1. 先统计受影响记录数和抽样 URL，确认替换后在 `jiji1.tv` 均能访问。
2. 仅替换**主机名完全等于** `test.jiji8.cc` 且路径以 `/upload/` 开头的记录。
3. 不对标题、正文、播放地址、查询参数中的普通文本做全局字符串替换。
4. 小批量执行并验证后再全量；记录迁移时间、数量和回滚 SQL。

不要在前端项目中执行数据库更新，也不要基于未知表名直接运行 SQL。

## 5. 前端改造：集中规范化图片 URL

### 5.1 修改范围

主改动文件：`src/api/core/image.ts`。

该文件已被首页广告、标签、视频详情推荐等多处复用，集中修复可保证同一历史图片 URL 在所有页面得到一致处理。不要只在 `HomeView.vue` 中写 `replace()`，否则搜索、详情推荐等入口仍会留下同类问题。

当前工作区该文件有未提交改动。DeepSeek 必须把下述逻辑合并到现状中，而不是先还原成 Git 基线。

### 5.2 规范化规则

1. 空值继续返回现有按类型区分的占位图。
2. 相对路径 `/upload/...` 或 `upload/...` 统一拼到正式图片源 `https://jiji1.tv`。
3. 只对白名单旧主机名 `test.jiji8.cc`（必要时包含 `www.test.jiji8.cc`）替换 host 为 `jiji1.tv`。
4. 其他合法外部图片地址（广告 CDN 等）必须原样返回，不能强制替换域名。
5. 支持协议相对地址 `//test.jiji8.cc/...`，按 HTTPS 处理。
6. 使用浏览器标准 `URL` 对象修改 `hostname`，**不能**用对整段字符串无条件 `.replace('test.jiji8.cc', 'jiji1.tv')`，以免误改查询参数、路径或非图片 URL。
7. URL 解析异常时，不抛异常影响列表渲染；返回原值或该类型占位图，并用开发环境日志记录即可。

建议实现（名称可按项目规范微调）：

```ts
const CANONICAL_IMAGE_ORIGIN = 'https://jiji1.tv'
const LEGACY_IMAGE_HOSTS = new Set(['test.jiji8.cc', 'www.test.jiji8.cc'])

const isHttpUrl = (value: string) => /^https?:\/\//i.test(value)

const createPlaceholder = (type: 'video' | 'ad' | 'tag' | 'avatar') => {
  const placeholders = {
    video: 'https://via.placeholder.com/300x400/222/999?text=无封面',
    ad: 'https://via.placeholder.com/750x200/222/999?text=广告加载中',
    tag: 'https://via.placeholder.com/150x200/222/999?text=分类',
    avatar: 'https://via.placeholder.com/100/333/ccc?text=User',
  }
  return placeholders[type]
}

export const getFullImageUrl = (
  path: string | undefined | null,
  type: 'video' | 'ad' | 'tag' | 'avatar' = 'video',
): string => {
  const raw = path?.trim()
  if (!raw) return createPlaceholder(type)

  // 保留 data URI，避免破坏未来可能存在的内嵌图片。
  if (raw.startsWith('data:image/')) return raw

  const normalizedRaw = raw.startsWith('//') ? `https:${raw}` : raw

  if (isHttpUrl(normalizedRaw)) {
    try {
      const url = new URL(normalizedRaw)
      if (LEGACY_IMAGE_HOSTS.has(url.hostname.toLowerCase())) {
        url.protocol = 'https:'
        url.hostname = 'jiji1.tv'
        url.port = ''
      }
      return url.toString()
    } catch {
      return createPlaceholder(type)
    }
  }

  const base = CANONICAL_IMAGE_ORIGIN.replace(/\/+$/, '')
  const relativePath = normalizedRaw.startsWith('/') ? normalizedRaw : `/${normalizedRaw}`
  return `${base}${relativePath}`
}
```

关于这段代码：

- 样例中的 URL 会被转换为 `https://jiji1.tv/upload/vod/20260712-1/14bda738f4d86aeb7e88004fd5d7f3f9.webp`，已验证它返回 `200 image/webp`。
- 如果项目团队要求图片必须经同源 `/api` 代理，可把旧域名的结果转换为 `/api${url.pathname}${url.search}${url.hash}`，但必须在开发 Vite 代理和生产 Nginx 中确认 `/api/upload/...` 均透传 `image/webp`、缓存头与状态码。不能只改开发环境。
- `url.toString()` 会为纯域名补 `/`；这不是问题，因为下一步会通过候选选择过滤域名根路径，详见下一节。

### 5.3 在首页过滤无效封面候选项

修改 `src/views/HomeView.vue` 的 `processVideoData`。不要再只用简单的 `a || b || c` 来挑选封面；将根路径 URL 排除掉，避免样例中的 `vod_pic_thumb: 'https://test.jiji8.cc/'` 被用作图片。

建议新增纯函数：

```ts
const isMeaningfulCoverCandidate = (value: unknown): value is string => {
  if (typeof value !== 'string' || !value.trim()) return false

  const raw = value.trim()
  if (raw.startsWith('data:image/')) return true

  try {
    const url = new URL(raw.startsWith('//') ? `https:${raw}` : raw, 'https://jiji1.tv')
    return url.pathname !== '/' && url.pathname !== ''
  } catch {
    return false
  }
}

const resolveVideoCover = (item: ApiVideoItem): string => {
  const candidate = [item.vod_pic, item.vod_pic_thumb, item.coverUrl]
    .find(isMeaningfulCoverCandidate)

  return getFullImageUrl(candidate, 'video')
}
```

映射改为：

```ts
coverUrl: resolveVideoCover(item),
```

说明：

- 对本次样例，首选仍然是 `vod_pic`，但通过统一 URL 工具转到 `jiji1.tv`，因此能正常加载。
- 若未来 `vod_pic` 为空而 `vod_pic_thumb` 只是 `https://域名/`，该值会被跳过，继续尝试 `coverUrl`，最后使用占位图。
- 不要在 `processVideoData` 内发 `fetch`/`HEAD` 请求验证图片是否存在：会额外产生 N 个网络请求、可能被 CORS 阻断、放大列表切换开销。加载错误由 `<img>` 事件处理。

若搜索页、标签页等也各自实现了封面候选选择，DeepSeek 应把“跳过根路径 URL”的小函数抽到图片工具中复用，或逐页统一接入；至少不得让首页修好后搜索页仍把站点首页当封面。

## 6. 前端兜底 UI：图片失败不能只剩黑块

修改 `src/components/VideoList.vue`，作为数据仍可能存在单文件 404 时的最后一道兜底。该修改不替代 P0 的数据源修复。

### 要求

- `@error` 必须记录“本条视频 + 当前封面 URL”加载失败状态。
- 失败后移除失败的 `<img>`，改为展示带图标和“封面暂不可用”文字的占位层；不要无限重试同一个 URL。
- 状态 key 必须含 `item.id` 和 `item.coverUrl`，避免同一个视频修正 URL 后仍永久显示失败占位。
- 保持 `.video-cover` 的 `16 / 9` 比例、VIP/限免角标和标题布局不变。
- 不把错误的 `vod_pic_thumb`（页面 HTML）赋给 `img.src`，也不把一张外部占位图再作为失败时的唯一依赖。

示意代码：

```ts
const failedCoverKeys = ref(new Set<string>())

const getCoverKey = (item: VideoItem) => `${item.id ?? 'unknown'}:${item.coverUrl}`

const markCoverFailed = (item: VideoItem) => {
  const next = new Set(failedCoverKeys.value)
  next.add(getCoverKey(item))
  failedCoverKeys.value = next
}

const hasCoverFailed = (item: VideoItem) => failedCoverKeys.value.has(getCoverKey(item))
```

模板将原图片区域改为类似：

```vue
<div class="video-cover">
  <img
    v-if="!hasCoverFailed(item)"
    v-lazy="item.coverUrl"
    :alt="item.title || '视频封面'"
    @error="markCoverFailed(item)"
  />
  <div v-else class="video-cover-fallback" role="img" aria-label="封面暂不可用">
    <van-icon name="photo-o" size="28" />
    <span>封面暂不可用</span>
  </div>
  <!-- 保留既有 VIP、限免、时长角标 -->
</div>
```

`failedCoverKeys` 需要从 Vue 导入 `ref`。若 `VideoList` 同时渲染广告，应让广告走同一失败兜底，或给广告使用独立但等价的 key；不能因为广告分支遗漏 `@error`。
