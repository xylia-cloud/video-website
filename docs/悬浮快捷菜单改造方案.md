# 右下角悬浮快捷菜单改造方案

> 目标：把现有右下角“可拖拽客服图片”改为截图所示的固定橙色悬浮快捷菜单。收起时只显示一个橙色圆形和**向左箭头**；点击后按钮箭头变为**向上**，上方依次展开“大厅、充值、客服、视频”四个圆形入口及黑色标签。
>
> 本文用于交给 DeepSeek 实施。请在现有代码基础上做增量修改，不要覆盖工作区中其他未提交的改动。

## 1. 已确认的业务规则

| 菜单项（从上到下） | 图标 | 点击结果 | 实现方式 |
| --- | --- | --- | --- |
| 大厅 | 首页/房屋 | 进入游戏大厅 | `router.push({ name: 'game' })`，即 `/game` |
| 充值 | 钱包/卡包 | 进入充值页面 | `router.push({ name: 'recharge' })`，即 `/recharge` |
| 客服 | 耳机客服 | 直接弹出全屏客服页 | `openCustomerServiceModal()`；**不跳转 `/customer-service`，不回首页** |
| 视频 | 播放 | 返回视频首页 | `router.push({ name: 'home' })`，即 `/` |

充值路由已有 `requiresAuth: true`，不要在悬浮菜单中重复写登录校验或自行跳登录页，仍由 `src/router/index.ts` 的全局路由守卫统一处理。

当前 `src/views/CustomerServiceView.vue` 的行为是“打开全局客服弹窗后 `router.replace('/')`”。本次快捷菜单的客服入口**不要使用此路由**，而是直接调用 `openCustomerServiceModal()`，因此不会发生回首页的路由操作。`CustomerServiceView.vue`、`GlobalCustomerServiceModal.vue` 与其他页面已有的客服入口均不属于本次改动范围。

## 2. 现状与改造范围

当前组件为 `src/components/CustomerServiceButton.vue`，被 `src/App.vue` 全局挂载：

```vue
<CustomerServiceButton />
```

它目前存在以下行为：

- 显示 `@/assets/img/kefu.png` 图片；
- 支持鼠标和触摸拖拽；
- 把坐标写到 `localStorage.customerServiceButtonPosition`；
- 点击后只调用 `openCustomerServiceModal()`；
- 在 `/game-play` 或全局客服弹窗打开时隐藏。

本次只改造 `src/components/CustomerServiceButton.vue`，保留文件名和 `App.vue` 的引用，避免引入无关的全局挂载改动。

### 必须移除的旧逻辑

将以下内容从该组件删除：

- 客服图片 `<img src="@/assets/img/kefu.png" ...>`；
- `position`、`buttonStyle`、`isDragging`、`dragStart`、`hasMoved` 等拖拽状态；
- `touchstart/touchmove/touchend/mousedown` 及 document 鼠标监听；
- 读取或写入 `customerServiceButtonPosition` 的 `localStorage` 逻辑；
- 仅能打开客服的旧 `handleClick`。

不要为了“清理”而在运行时调用 `localStorage.removeItem('customerServiceButtonPosition')`；旧键不再读取即可，避免给用户增加无意义的本地写入。

保留以下可见性规则并改名为 `shouldShowMenu`：

```ts
const shouldShowMenu = computed(() => {
  return route.name !== 'game-play' && !isCustomerServiceModalOpen.value
})
```

这样游戏全屏页与已弹出的客服页不会叠加快捷按钮。若客服弹窗打开，菜单必须自动收起。

## 3. 视觉与布局规格

以需求截图为准，使用项目已有 Vant 图标，不新增图片资源。

### 3.1 固定位置与尺寸

- 菜单固定在右下角，不可拖拽。
- 容器定位：`position: fixed`，`right: 16px`，`bottom: calc(64px + env(safe-area-inset-bottom))`。
  - `64px` 用于避开项目中高度为 `50px` 的 `BottomTabbar`；页面没有底栏时保留该位置，不按页面动态跳动。
  - 窄屏可把 `right` 调整为 `12px`，但不得贴边或被安全区遮挡。
- 主开关圆形：`88px × 88px`；快捷项圆形：`68px × 68px`。
- 快捷项与下一个快捷项的垂直间距：`14px`；最下方“视频”与主开关间距：`16px`。
- 菜单标签在圆形按钮右侧，宽 `90px`、高 `38px`、圆角 `20px`，与圆形按钮垂直居中。
- 容器 `z-index: 1000`，高于 `BottomTabbar`（其 z-index 为 999），低于全局客服弹窗（10050）和对话框。

### 3.2 颜色、图标与状态

| 元素 | 规格 |
| --- | --- |
| 主开关与快捷圆形 | 橙色 `#ff9500`；可用 `#ff8a00 → #ff9500` 的轻微渐变，但不要使用图片背景 |
| 图标 | 白色，主开关约 `36px`，快捷项约 `30px` |
| 标签底色 | `rgba(0, 0, 0, 0.86)` |
| 标签文字 | 白色，`22px`，字重 500；文本为“大厅 / 充值 / 客服 / 视频” |
| 阴影 | `0 6px 16px rgba(0, 0, 0, 0.28)`，不要产生明显发光效果 |
| 收起主开关 | Vant `arrow-left`，无菜单项 |
| 展开主开关 | Vant `arrow-up`，显示四项菜单 |
| 四个快捷项图标 | 依次优先使用 `home-o`、`gold-coin-o`（或项目中可用的等价钱包图标）、`service-o`、`play-circle-o` |

项目使用 Vant `4.9.19`。实施时先在本地确认所选图标名称可用；若某个名称不存在，只替换为同语义的 Vant 内置图标，不要临时引入第三方图标库或 SVG 图片。

### 3.3 DOM 排列（确保与截图一致）

菜单视觉从上到下必须是：**大厅 → 充值 → 客服 → 视频 → 主开关**。

主开关靠右；每行快捷项的橙色圆形在左、黑色文字标签在右。菜单整体的标签右边缘与主开关右边缘对齐，避免标签超出屏幕。

建议结构：

```vue
<div v-if="shouldShowMenu" class="floating-quick-menu">
  <TransitionGroup name="quick-menu" tag="div" class="quick-menu-list">
    <div v-for="item in menuItems" v-if="isExpanded" :key="item.key" class="quick-menu-item">
      <button type="button" class="quick-menu-action" @click="handleMenuAction(item)">
        <van-icon :name="item.icon" />
      </button>
      <span class="quick-menu-label">{{ item.label }}</span>
    </div>
  </TransitionGroup>

  <button
    type="button"
    class="quick-menu-toggle"
    :aria-expanded="isExpanded"
    :aria-label="isExpanded ? '收起快捷菜单' : '展开快捷菜单'"
    @click="toggleMenu"
  >
    <van-icon :name="isExpanded ? 'arrow-up' : 'arrow-left'" />
  </button>
</div>
```

注意：`TransitionGroup` 的子项必须随 `isExpanded` 真正进入和离开 DOM，才能执行进/离场动画；不要把 `v-show` 当作离场动画方案。上面的 `v-if` 条件只依赖外层状态、无需读取 `item`，在本项目的 Vue 3 模板中可用。若 DeepSeek 的模板规范不允许同节点写 `v-for` 与 `v-if`，可为每个 `item` 增加一层带 `:key="item.key"` 的 `<template v-for>`，将 `v-if="isExpanded"` 放到实际的 `.quick-menu-item` 上。

## 4. 交互要求

### 4.1 展开和收起

1. 初始状态 `isExpanded = false`，只显示主开关和向左箭头。
2. 点击主开关后置为 `true`：箭头切换为向上，菜单从主开关上方向上展开。
3. 再次点击主开关后置为 `false`：菜单反向收起，箭头恢复向左。
4. 点击任一菜单项时，先立即收起菜单，再执行跳转或打开客服弹窗，避免路由/弹窗切换时残留展开状态。
5. 任意路由切换后自动收起：监听 `route.fullPath`，将 `isExpanded.value = false`。
6. 全局客服弹窗变为打开状态时，自动收起；其打开期间整个快捷菜单隐藏。
7. PC 端点击圆形和标签均只触发所在菜单项；移动端连续快速点击不得造成重复 `router.push` 或重复打开客服弹窗。

本需求不要求“点击页面空白处收起”。如实现该能力，必须只在菜单展开时监听，并在组件卸载时移除监听；不得拦截视频播放、弹窗或页面滚动事件。为控制改动范围，推荐本次不做空白处收起。

### 4.2 动画

- 展开总时长控制在 `220–280ms`，使用 `ease-out`；收起用 `180–220ms`，使用 `ease-in`。
- 快捷项的进入效果：`opacity: 0 → 1`，`transform: translateY(12px) scale(0.9) → translateY(0) scale(1)`。
- 退出动画反向执行。
- 按距离主开关由近到远依次出现：视频 → 客服 → 充值 → 大厅；每项间隔 `35–45ms`。即使动画进行中也不应导致布局跳动。
- 主开关按压反馈可使用 `transform: scale(0.96)`，不要再给整个容器添加 `:hover { transform: scale(...) }`，否则会改变已展开菜单坐标。
- 用户系统启用“减少动态效果”（`prefers-reduced-motion: reduce`）时禁用位移和错峰动画，仅保留即时显隐。

### 4.3 可访问性与点击安全

- 所有可点击元素使用 `<button type="button">`，不要用普通 `div` 绑定点击。
- 主开关必须带 `aria-expanded` 和动态 `aria-label`。
- 每个快捷项按钮须有准确的 `aria-label`，如“进入游戏大厅”“进入充值页面”“打开在线客服”“返回视频首页”。标签文字仅用于视觉展示。
- 图标加 `aria-hidden="true"`，由按钮文本/`aria-label` 提供含义。
- `:focus-visible` 添加白色或深色描边，不能只依赖鼠标 hover。
- 当主开关获得焦点时按 `Enter`、`Space` 应可展开；展开后按 `Escape` 收起（在 `onMounted` 绑定 `keydown`，`onBeforeUnmount` 清理）。

## 5. 推荐脚本实现

在 `src/components/CustomerServiceButton.vue` 的 `<script setup lang="ts">` 中按以下思路重写。下面代码是实现骨架，DeepSeek 需要结合实际模板和样式完整落地。

```ts
import { computed, onBeforeUnmount, onMounted, ref, watch } from 'vue'
import { useRoute, useRouter } from 'vue-router'
import { isCustomerServiceModalOpen, openCustomerServiceModal } from '@/utils/customerService'

type QuickMenuAction = 'route' | 'customer-service'

interface QuickMenuItem {
  key: 'game' | 'recharge' | 'customer-service' | 'home'
  label: '大厅' | '充值' | '客服' | '视频'
  icon: string
  ariaLabel: string
  action: QuickMenuAction
  routeName?: 'game' | 'recharge' | 'home'
}

const route = useRoute()
const router = useRouter()
const isExpanded = ref(false)
const isNavigating = ref(false)

const menuItems: QuickMenuItem[] = [
  { key: 'game', label: '大厅', icon: 'home-o', ariaLabel: '进入游戏大厅', action: 'route', routeName: 'game' },
  { key: 'recharge', label: '充值', icon: 'gold-coin-o', ariaLabel: '进入充值页面', action: 'route', routeName: 'recharge' },
  { key: 'customer-service', label: '客服', icon: 'service-o', ariaLabel: '打开在线客服', action: 'customer-service' },
  { key: 'home', label: '视频', icon: 'play-circle-o', ariaLabel: '返回视频首页', action: 'route', routeName: 'home' },
]

const shouldShowMenu = computed(() => {
  return route.name !== 'game-play' && !isCustomerServiceModalOpen.value
})

const toggleMenu = () => {
  isExpanded.value = !isExpanded.value
}

const closeMenu = () => {
  isExpanded.value = false
}

const handleMenuAction = async (item: QuickMenuItem) => {
  if (isNavigating.value) return

  closeMenu()

  if (item.action === 'customer-service') {
    // 已确认：直接弹出客服页，不经过 /customer-service，也不回首页。
    openCustomerServiceModal()
    return
  }

  if (!item.routeName || route.name === item.routeName) return

  isNavigating.value = true
  try {
    await router.push({ name: item.routeName })
  } finally {
    isNavigating.value = false
  }
}

const handleKeydown = (event: KeyboardEvent) => {
  if (event.key === 'Escape' && isExpanded.value) closeMenu()
}

watch(() => route.fullPath, closeMenu)
watch(isCustomerServiceModalOpen, (isOpen) => {
  if (isOpen) closeMenu()
})

onMounted(() => window.addEventListener('keydown', handleKeydown))
onBeforeUnmount(() => window.removeEventListener('keydown', handleKeydown))
```

实现注意事项：

- 不要用 `router.push('/customer-service')`，否则会进入现有 `CustomerServiceView` 的“打开弹窗后回首页”流程，违背本次已确认的客服行为。
- 菜单触发路由时，`router.push` 可能被守卫重定向到登录页；这是充值未登录时的正确结果，`finally` 必须正常恢复 `isNavigating`。
- 当前页面已是目标页面时只收起菜单，不要重复导航，也不要报 `NavigationDuplicated`。
- `isNavigating` 只防重复路由点击；客服弹窗由 `isCustomerServiceModalOpen` 控制，不应在其打开期间再次显示按钮。

## 8. 回归验收清单

### 8.1 手工验收

1. 在首页、直播、游戏大厅、个人中心、视频详情等页面，右下角均显示橙色圆形主开关，且位于底部导航上方。
2. 初始状态箭头向左，四个快捷项完全不可见且不可点击。
3. 点击主开关后，箭头变为向上；菜单自下向上依次显示视频、客服、充值、大厅，最终视觉顺序必须为大厅、充值、客服、视频。
4. 再次点击主开关，四项按反向动画收起，箭头回到向左。
5. 点击“大厅”进入 `/game`；点击“视频”进入 `/`；在已经位于目标页时只收起菜单。
6. 点击“充值”进入 `/recharge`；未登录时由现有路由守卫进入登录页，并保留 `redirect=/recharge`。
7. 点击“客服”后直接出现现有全屏客服弹窗；地址栏不经过 `/customer-service`，底层页面不被重定向为首页。
8. 在客服弹窗可见期间，快捷菜单不显示；关闭客服弹窗后，在原来的底层页面恢复显示且为收起状态。
9. 进入 `/game-play` 时快捷菜单不显示；离开后按正常规则出现。
10. 刷新页面和跨页面跳转后，菜单始终固定右下角，不恢复任何旧拖拽坐标。
11. 在 iPhone 刘海/底部安全区、Android、375px 与 768px 宽度下，标签没有被右侧裁切，主开关不压住底部导航。
12. 鼠标、触摸、键盘均能操作；焦点样式可见，展开时按 `Escape` 能收起。
