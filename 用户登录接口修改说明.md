# 用户登录和注册接口修改说明

## 修改概述

将项目中的用户登录和注册接口从原有地址修改为新的接口地址：

- 登录接口：`http://live.88tv.co/appapi/?service=login.userLogin`
- 注册接口：`http://live.88tv.co/appapi/?service=login.userReg`

## 修改文件

### 1. `src/utils/config.ts`

- 新增 `USER_API_BASE_URL` 配置：`'http://live.88tv.co/appapi'`
- 用于存放用户相关接口的基础URL

### 2. `src/api/fetch-api.ts`

- 导入新的 `USER_API_BASE_URL` 配置
- 修改 `userLogin` 函数实现：
  - 参数格式：支持 `user_name`、`user_pwd` 和可选的 `country_code`（默认86）
  - 请求方式：使用 URLSearchParams 构建查询参数
  - 接口地址：`${USER_API_BASE_URL}/?service=login.userLogin`
  - 参数映射：
    - `user_name` → `user_login`
    - `user_pwd` → `user_pass`
    - `country_code` → `country_code`（默认86）
- 修改 `registerUser` 函数实现：
  - 参数格式：支持 `user_name`、`user_pwd`、`user_pwd2`
  - 请求方式：使用 URLSearchParams 构建查询参数
  - 接口地址：`${USER_API_BASE_URL}/?service=login.userReg`
  - 参数映射：
    - `user_name` → `user_login`
    - `user_pwd` → `user_pass`
    - `user_pwd2` → `user_pass2`
    - 固定 `country_code` = 86

## 接口返回格式适配

### 新接口返回格式

**登录接口返回格式：**

```json
{
  "ret": 200,
  "data": {
    "code": 0,
    "msg": "",
    "info": {
      "id": "1743",
      "user_nicename": "手机用户3456",
      "avatar": "http://live.88tv.co/upload/admin/...",
      "score": "10",
      "level": "1",
      "token": "69873a52779cee6798cbcd3206f341e6",
      "rec_code": "ZT72KW"
    }
  }
}
```

**注册接口返回格式：**

```json
{
  "ret": 200,
  "data": {
    "code": 0,
    "msg": "操作成功",
    "info": [1]
  },
  "msg": ""
}
```

### 数据字段映射

| 新接口字段          | 原字段名       | 说明                   |
| ------------------- | -------------- | ---------------------- |
| info.id             | user_id        | 用户ID                 |
| params.user_name    | user_name      | 用户名（使用登录参数） |
| info.token          | token          | 登录令牌               |
| info.avatar         | user_portrait  | 用户头像               |
| info.score          | user_points    | 用户积分               |
| info.level          | group_id       | 用户等级               |
| `等级${info.level}` | group_name     | 等级名称               |
| info.user_nicename  | user_nick_name | 用户昵称               |
| info.rec_code       | rec_code       | 推荐码                 |

### 成功/失败判断

- **登录成功**：`ret === 200 && data.code === 0`，返回完整用户信息
- **注册成功**：`ret === 200 && data.code === 0`，只返回成功标识 `[1]`
- **失败**：其他情况，使用 `data.msg` 作为错误信息

## 注意事项

- 新的登录和注册接口都使用POST方法，但参数通过URL查询字符串传递
- 已适配新接口的返回格式，自动转换为原有的数据结构
- 保持了原有的错误处理和用户信息存储逻辑
- 注册接口固定使用国家代号86
- 注册成功后只返回简单的成功标识，需要调用登录接口获取完整用户信息
- 注册页面已实现注册成功后自动登录功能
- 如果需要对接其他用户相关接口，可以使用相同的 `USER_API_BASE_URL` 配置

## 退出登录接口修改（新增）

### 接口信息

- **原接口**: `${API_PREFIX}/index.php/user/login` (POST请求)
- **新接口**: `http://live.88tv.co/appapi/?service=login.logout` (POST请求)

### 参数说明

| 参数名  | 参数类型 | 是否必填 | 说明示例             |
| ------- | -------- | -------- | -------------------- |
| service | string   | 是       | 固定值：login.logout |
| uid     | int      | 是       | 用户ID               |
| token   | string   | 是       | 用户token            |

### 返回格式

**成功退出**:

```json
{
  "ret": 200,
  "data": {
    "code": 0,
    "msg": "操作成功"
  }
}
```

**token失效**:

```json
{
  "ret": 200,
  "data": {
    "code": 700,
    "msg": "您的登陆状态失效，请重新登陆！"
  }
}
```

### 实现说明

- **请求方式**: POST请求，但参数通过URL查询字符串传递（与登录和注册接口保持一致）
- 退出登录函数会从本地存储获取用户ID和token
- 无论接口调用成功或失败，都会清除本地所有缓存数据
- 即使返回token失效（code: 700），也视为正常退出，清除本地缓存
- 保持原有的退出登录流程和用户体验

### 调用示例

```javascript
// 实际请求URL
http://live.88tv.co/appapi/?service=login.logout&uid=1743&token=69873a52779cee6798cbcd3206f341e6

// 请求方式：POST
// 请求头：
{
  "Accept": "application/json, text/plain, */*",
  "User-Agent": "Mozilla/5.0 (iPhone; CPU iPhone OS 16_6 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/16.6 Mobile/15E148 Safari/604.1"
}
```

## 用户编辑资料接口修改说明

### 3. 获取用户基本信息接口

- **接口地址**：`user.getBaseInfo`
- **请求方式**：POST
- **请求URL**：`http://live.88tv.co/appapi/?service=user.getBaseInfo&uid={用户ID}&token={用户token}`

| 参数名  | 参数类型 | 是否必填 | 说明             |
| ------- | -------- | -------- | ---------------- |
| service | string   | 是       | user.getBaseInfo |
| uid     | int      | 是       | 用户ID           |
| token   | string   | 是       | 用户token        |

### 4. 修改用户头像接口

- **接口地址**：`user.updateAvatar`
- **请求方式**：POST
- **请求URL**：`http://live.88tv.co/appapi/?service=user.updateAvatar&uid={用户ID}&token={用户token}`

| 参数名  | 参数类型 | 是否必填 | 说明                         |
| ------- | -------- | -------- | ---------------------------- |
| service | string   | 是       | user.updateAvatar            |
| uid     | int      | 是       | 用户ID                       |
| token   | string   | 是       | 用户token                    |
| file    | string   | 否       | 头像图片（支持文件和base64） |

**实现说明**：

- 支持两种上传方式：文件对象和base64编码
- 基础参数（service、uid、token）通过URL查询参数传递
- 文件数据通过FormData传递
- 文件参数名称：`file`
- 文件上传：`FormData.append('file', file)`
- Base64上传：`FormData.append('file', base64String)`
- 图片直接走本地上传返回头像地址

### 5. 修改用户信息接口

- **接口地址**：`user.updateFields`
- **请求方式**：POST
- **请求URL**：`http://live.88tv.co/appapi/?service=user.updateFields&uid={用户ID}&token={用户token}&fields={JSON字符串}`

| 参数名  | 参数类型 | 是否必填 | 说明                     |
| ------- | -------- | -------- | ------------------------ |
| service | string   | 是       | user.updateFields        |
| uid     | int      | 是       | 用户ID                   |
| token   | string   | 是       | 用户token                |
| fields  | string   | 是       | 修改信息，JSON字符串格式 |

**fields JSON格式示例**：

```json
{
  "user_nicename": "用户昵称",
  "avatar": "头像地址（通过单独接口上传）",
  "sex": "性别（0保密，1男，2女）",
  "signature": "个性签名",
  "birthday": "生日（YYYY-MM-DD格式）",
  "location": "位置",
  "province": "省份",
  "city": "城市"
}
```

**实际修改字段**：只修改 `user_nicename`, `avatar`, `sex`, `birthday` 这几项，其他字段为空值。

**性别值说明**：

- 0: 保密
- 1: 男
- 2: 女

**注意事项**：

- 头像通过单独的 `user.updateAvatar` 接口上传
- 生日格式为 YYYY-MM-DD（例如：2024-01-15）
- 其他字段（signature, location, province, city）暂时保持为空字符串

### 6. 修改用户密码接口

- **接口地址**：`user.updatePass`
- **请求方式**：POST
- **请求URL**：`http://live.88tv.co/appapi/?service=user.updatePass&uid={用户ID}&token={用户token}&oldpass={旧密码}&pass={新密码}&pass2={确认密码}`

| 参数名  | 参数类型 | 是否必填 | 说明            |
| ------- | -------- | -------- | --------------- |
| service | string   | 是       | user.updatePass |
| uid     | int      | 是       | 用户ID          |
| token   | string   | 是       | 用户token       |
| oldpass | string   | 是       | 旧密码          |
| pass    | string   | 是       | 新密码          |
| pass2   | string   | 是       | 确认密码        |

## 接口实现特点

### 统一的返回格式转换

所有新接口都会将返回格式转换为前端兼容的格式：

- 成功：`{ code: 1, data: {...}, msg: "成功信息" }`
- 失败：`{ code: 0, data: null, msg: "错误信息" }`

### 自动更新本地用户信息

- 头像上传成功后自动更新本地存储的`user_portrait`字段
- 个人信息修改成功后自动更新本地存储的对应字段
- 确保前端UI能够实时反映最新的用户信息

### 错误处理

- 参数验证：检查必填参数是否为空
- 登录状态检查：确保用户已登录且token有效
- 网络错误处理：提供友好的错误提示信息
- 响应格式检查：验证接口返回的数据格式
