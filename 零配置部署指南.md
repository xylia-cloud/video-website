# 🚀 零配置部署指南

## ✨ 核心特性

**一行代码，智能适配所有域名** - 无需配置，支持任意域名自动部署

## 🎯 工作原理

```typescript
// 🌐 零配置API地址 - 6行代码解决所有域名问题
const isLocal = (host) => /localhost|127\.0\.0\.1|192\.168\.|10\.|172\./.test(host)

export const BASE_URL = import.meta.env.DEV
  ? '/api'
  : import.meta.env.VITE_API_BASE_URL ||
    (globalThis.location && !isLocal(location.hostname)
      ? `${location.protocol}//${location.hostname}`
      : 'https://jiji1.tv')
```

## 📋 适配规则

| 环境        | 使用地址            | 说明                 |
| ----------- | ------------------- | -------------------- |
| 🔧 开发环境 | `/api`              | 代理到目标服务器     |
| ⚙️ 环境变量 | `VITE_API_BASE_URL` | 手动指定API地址      |
| 🌐 自动检测 | 当前域名            | **智能适配任意域名** |
| 🏠 兜底方案 | `jiji1.tv`          | 安全回退             |

## 🚀 部署方式

### 零配置部署（推荐）

```bash
npm run build
# 部署到任意域名，自动适配 ✨
```

**支持域名示例：**

- `jiji8.cc` ✅
- `newdomain.com` ✅
- `example.org` ✅
- `any-domain.net` ✅

### 自定义API地址

```bash
# 方式1：环境变量
echo "VITE_API_BASE_URL=https://api.example.com" > .env.production

# 方式2：构建时指定
VITE_API_BASE_URL=https://api.example.com npm run build
```

## 🎯 使用场景

### ✅ 完美支持

- **域名迁移**：从 `jiji8.cc` 换到 `newsite.com`
- **多镜像站**：同一代码部署到多个域名
- **测试环境**：快速部署到测试域名

### ⚠️ 特殊情况

- **API分离**：网站和API使用不同域名时，设置环境变量
- **内网部署**：使用内网IP时，建议设置环境变量

## 🔧 故障排除

### 问题：API地址不正确

1. 打开浏览器控制台
2. 检查 `BASE_URL` 的值
3. 确认域名是否正确解析

### 问题：本地开发问题

1. 确认使用 `npm run dev`
2. 检查 `vite.config.ts` 代理配置
