# 🚨 301重定向问题修复

## 📋 问题描述

从Network图片可以看到：

- 请求从 `localhost:3000/api/ads.html` 返回 `301 Moved Permanently`
- 然后重定向到 `https://jiji1.tv/ads.html` 导致CORS错误

## 🔍 问题分析

### 问题根源

1. **Vite代理配置问题**：代理可能没有正确处理请求
2. **目标服务器重定向**：jiji1.tv 可能对某些请求返回301
3. **开发环境配置错误**：BASE_URL 可能没有正确解析为 `/api`

## ✅ 应急修复

### 修复1: 强制开发环境使用API_PREFIX

```javascript
// 在 fetchAds 函数中添加应急修复
const apiUrl = import.meta.env.DEV ? API_PREFIX : BASE_URL
const fullUrl = `${apiUrl}/index.php/ajax/ads.html?${queryParams.toString()}`
```

### 修复2: 增强调试信息

```javascript
console.log('🔧 当前BASE_URL配置:', BASE_URL)
console.log('🔧 当前环境:', import.meta.env.DEV ? '开发环境' : '生产环境')
console.log('🔧 import.meta.env.DEV值:', import.meta.env.DEV)
console.log('🚑 实际使用的API地址:', apiUrl)
console.log('🌐 请求URL:', fullUrl)
```

### 修复3: 启用代理调试日志

```javascript
// vite.config.ts 中启用详细日志
proxy.on('proxyReq', (proxyReq, req) => {
  console.log('🚀 发送代理请求:', req.method, req.url)
  console.log('🎯 目标地址:', proxyReq.path)
})
proxy.on('proxyRes', (proxyRes, req) => {
  const statusCode = proxyRes.statusCode || -1
  console.log('📥 收到代理响应:', statusCode, req.url)
  if (statusCode === 301 || statusCode === 302) {
    console.warn('⚠️ 检测到重定向，Location:', proxyRes.headers.location)
  }
})
```
