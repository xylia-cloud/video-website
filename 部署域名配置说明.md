# 部署域名配置说明

## 🌐 域名自动适配机制

本项目支持**动态域名检测**，可以自动适配任何部署域名，无需修改代码。

## 📋 配置优先级

系统按以下优先级自动选择API基础URL：

1. **开发环境** (`npm run dev`)

   - 使用 `/api` 代理到目标服务器
   - 通过 `vite.config.ts` 配置代理规则

2. **生产环境环境变量** (`VITE_API_BASE_URL`)

   - 最高优先级
   - 适用于需要固定API地址的场景

3. **生产环境自动检测**

   - 自动使用当前访问的域名
   - 支持任意域名（jiji8.cc、jiji1.tv、example.com等）
   - 排除开发环境地址（localhost、127.0.0.1、内网IP）

4. **兜底方案**
   - `https://jiji1.tv`
   - 防止配置失败的安全机制

## 🚀 部署方式

### 方式一：自动检测（推荐）

**无需任何配置**，直接部署到任何域名：

```bash
# 构建
npm run build

# 部署到任意域名
# 系统会自动检测并使用当前域名作为API地址
```

**支持的域名示例：**

- `https://jiji8.cc` → API地址: `https://jiji8.cc`
- `https://newdomain.com` → API地址: `https://newdomain.com`
- `https://example.org` → API地址: `https://example.org`

### 方式二：环境变量配置

如果需要API地址与访问域名不同：

1. 创建 `.env.production` 文件：

```env
VITE_API_BASE_URL=https://api.yourdomain.com
```

2. 构建部署：

```bash
npm run build
```

### 方式三：构建时指定

```bash
# 临时指定API地址
VITE_API_BASE_URL=https://api.example.com npm run build
```

## 📝 常见场景

### 场景1：域名变更

```
旧域名: jiji8.cc
新域名: newsite.com
操作: 无需修改代码，直接部署到新域名即可
```

### 场景2：多域名部署

```
主站: main.com
镜像站1: mirror1.com
镜像站2: mirror2.com
操作: 同一套代码可部署到所有域名
```

### 场景3：API分离

```
网站域名: www.example.com
API域名: api.example.com
操作: 设置 VITE_API_BASE_URL=https://api.example.com
```

## ⚠️ 注意事项

1. **CORS配置**：确保目标API服务器支持当前域名的跨域请求
2. **HTTPS**：生产环境建议使用HTTPS协议
3. **防火墙**：确保服务器能访问API地址
4. **域名解析**：确保域名正确解析到服务器IP

## 🛠️ 故障排除

### 问题：API请求失败

1. 打开浏览器开发者工具
2. 查看Console中的域名检测日志
3. 检查Network中的请求地址是否正确
4. 确认API服务器是否可访问

### 问题：域名检测错误

1. 确认不是在localhost等开发环境访问
2. 检查域名是否正确配置
3. 可通过环境变量强制指定API地址
