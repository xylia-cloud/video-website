# 充值订单接口实现说明

## 接口信息

- **接口地址**: `charge.getchargeorders`
- **请求方式**: POST
- **完整URL**: `http://live.88tv.co/appapi/?service=charge.getchargeorders`

## 请求参数

| 参数名      | 类型   | 是否必填 | 说明                                            |
| ----------- | ------ | -------- | ----------------------------------------------- |
| uid         | int    | 是       | 用户ID                                          |
| token       | string | 是       | 用户token                                       |
| qudaoid     | int    | 是       | 支付渠道 (2:虚拟币; 3:银行卡; 4:微信; 5:支付宝) |
| money       | string | 是       | 金额                                            |
| paytypecode | string | 是       | 支付通道代码                                    |
| moneylistid | int    | 是       | 充值规则中的moneylist的id                       |

## 返回数据示例

```json
{
```

### 2. 前端调用逻辑 (`src/views/RechargeView.vue`)

#### 参数获取策略：

- **qudaoid**: 从用户选择的支付渠道映射获取
- **money**: 从用户选择的金额项获取
- **paytypecode**: 根据渠道类型动态获取
  - 虚拟币: 从充值规则的 `accountInfo.bankare` 获取
  - 微信/支付宝: 从选中的子渠道 `paycode` 获取
- **moneylistid**: 从用户选择的金额项 `id` 获取

#### 支付流程：

1. **参数验证**: 检查必需的选择项
2. **创建订单**: 调用 `createChargeOrder` 接口
3. **处理结果**:
   - 成功且有支付链接: 弹窗确认后打开支付页面
   - 成功但无支付链接: 显示操作成功提示
   - 失败: 显示错误信息

#### 用户体验优化：

- **防重复提交**: 使用 `isCreatingOrder` 状态控制
- **加载提示**: 按钮显示加载动画
- **错误处理**: 友好的错误提示信息
- **分渠道处理**: 虚拟币和其他支付方式的不同提示

## 支付通道代码 (paytypecode) 获取规则

### 虚拟币 (qudaoid = 2)

```typescript
paytypecode = chargeRulesDisplay.value?.accountInfo?.bankare || 'USDT-TRC20'
```

### 微信/支付宝 (qudaoid = 4/5)

```typescript
paytypecode = selectedSubChannel.value.paycode || ''
```

## 注意事项

- 虚拟币支付通常需要手动转账，可能不返回 `purl`
- 微信/支付宝支付会返回支付链接用于跳转
- 订单创建成功后应该更新用户余额（可能需要轮询或WebSocket）
